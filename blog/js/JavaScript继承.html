# JavaScript继承

TIP

JavaScript没有类的概念，父子对象之间全靠”原型链“（prototype chain）实现继承。ES5包含6中继承方式，各有缺点；ES6提供了class语法糖，是一种基于ES5寄生组合继承方式的一种实现。 构造函数(Constructor)具备一个prototype属性，其指向的对象中包含所有实例共享的属性和方法

# 原型链继承

以父类的实例作为子类的原型

function Parent(){
    this.label = 'label';
    this.list = [1];
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function Child(name){
}
Child.prototype = new Parent();
//实例化
const child1 = new Child();
const child2 = new Child();

child1.list.push(2);
//属性共享
child2.list;//[1, 2]

优点：

  • 继承了父类原型的属性和方法

缺点：

  • 父类无法接受子类的动态参数进行构建；
  • 仍然在原型上创建了属性，此时，多个子类的实例将共享同一个父类的属性，也就无法实现多继承

# 构造函数继承

在子类Constructor中以子类构造函数为执行上下文call父类Constructor

function Parent(label, list){
    this.label = label;
    this.list = list;
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function Child(label, list){
    Parent.call(this, label, list);
    this.age = 3;
}
//实例化
const child1 = new Child('child1', []);
const child2 = new Child('child2', [1,2]);

//属性地址不共享
child1.list.push(2);
child1.list;//[2]
child2.list;//[1,2]

//原型方法丢失
child1.showLabel();//Uncaught TypeError: child1.showLabel is not a function

优点：

  • 可以向父类传参实现动态构建
  • 可以多继承

缺点：

  • 没有继承父类原型上的方法

# 组合继承

原型链继承和借用构造函数继承的组合继承方式

function Parent(){
    this.label = 'label';
    this.list = [1];
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function Child(...args){
    Parent.call(this, args);
}
Child.prototype = new Parent();
//实例化
const child1 = new Child();
const child2 = new Child();

优点：

  • 可以向父类传参实现动态构建
  • 可以多继承
  • 可以继承父类构造函数和原型上的属性和方法

缺点：

  • 每次实例化子类都会执行父类构造方法两次，耗内存
  • 子类的构造函数会代替原型上的那个父类构造函数

# 原型式继承

基于Object.create()实现。用一个函数包装一个对象，然后返回这个函数的调用（相当于复制一个父类实例对象），这个函数就变成了个可以随意增添属性的实例或对象。

function Parent(){
    this.label = 'label';
    this.list = [1];
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function Child(proto){
    function F(){};
    F.prototype = proto;
    return new F();
}
const ParentInst = new Parent();
//实例化
const child1 = new Child(ParentInst);
const child2 = new Child(ParentInst);

优点：

  • 可以多继承
  • 可以继承父类构造函数和原型上的属性和方法

缺点：

  • 子类的实例属性在构造之后添加
  • 子类实例化时不能动态传参

# 寄生继承

在原型式继承的基础上包裹一层传参函数

function Parent(){
    this.label = 'label';
    this.list = [1];
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function ChildContet(proto){
    function F(){};
    F.prototype = proto;
    return new F();
}
const ParentInst = new Parent();
function Child(args){
    const child1 = ChildContent(ParentInst);
    child1.param = args;
    return child1;
}

优点：

  • 可以多继承
  • 可以继承父类构造函数和原型上的属性和方法

缺点：

  • 复用性差（每一种实例的构造函数都有差异）

# 寄生组合继承

结合寄生继承和组合继承，在寄生继承中复制父类原型（区别于寄生继承的复制父类实例）

function Parent(){
    this.label = 'label';
    this.list = [1];
}
Parent.prototype.showLabel = function(){
    console.log(this.label);
}

function ChildPrototype(constructor){
    function F(){};
    F.prototype = constructor.prototype;
    return new F();
}

function Child(...args){
    Parent.call(this, args)
}
//修正Child的原型，继承父类原型的方法和属性
Child.prototype = new ChildPrototype();

const child1 = new Child();
const child2 = new Child();

优点：

  • 解决了组合继承的Parent构造函数执行两次的问题，同时解决了寄生继承的复用性差的问题
