# CSS篇（一）

# 盒模型

页面渲染时，dom 元素所采用的 布局模型。可通过box-sizing进行设置。根据计算宽高的区域可分为：

  • content-box (W3C 标准盒模型，宽高区域为conten)
  • border-box (IE 盒模型，宽高区域为content + padding + border)
  • padding-box（No Browser Support）
  • margin-box（No Browser Support）

# BFC

Block Format Context(块级格式化上下文)，是一个独立的渲染区域，让处于 BFC 内部的元素与外部的元素相互隔离，使内外元素的定位不会相互影响。

# 触发条件

  • 根元素
  • position: fixed 或 absolute
  • overflow: 非visiable
  • display: inline-block; table-cell; table-caption; flex
  • float元素

# BFC规则

  • 属于同一个 BFC 的两个相邻 Box 垂直排列，margin会发生重叠
  • BFC区域内float元素不会与其他内容重叠，如果有文字内容，则会环绕在浮动元素周围
  • 计算BFC高度浮动元素也参与计算
  • BFC子元素不会超出BFC区域

# 应用

  • 自适应两栏布局（float + overflow）
  • 防止margin重叠
  • 防止浮动元素覆盖其他元素
  • 清除浮动

# link和@import引入区别

  • link功能广泛，可以定义rel, RSS等功能；@import只能用来导入CSS；
  • link在页面加载的过程中同步导入，@import在页面加载完成后进行；
  • link支持通过JS动态导入，@import不能动态操作
  • @import只支持性IE5+

# display:none和 visibility :hidden区别

  • 前者使元素不占据渲染位置，后者仍然占据渲染位置，但隐藏不显示
  • visibility :hidden具备继承特性，父元素hidden不显示，子元素visible可以显示；display:none的子元素都无法显示；
  • display:none触发reflow; visibility :hidden触发repaint

# position: sticky

sticky兼容性不太好，特别是移动端

  • sticky不会触发BFC
  • z-index无效
  • 当父元素的height：100%时，页面滑动到一定高度之后sticky属性会失效
  • 父元素不能有overflow:hidden或者overflow:auto属性
  • 父元素高度不能低于sticky高度，必须指定top、bottom、left、right4个值之一

# flex布局

浏览器端除了IE，兼容性比较好；移动端几乎都支持

/*CSS属性*/
flex: flex-basis, flex-grow, flex-shrink的简写属性，用来规定弹性元素如何伸长或缩短去适应flex-box的可用空间;
flex-basis:指定了flex元素在主轴方向上的初始大小。如果不使用box-sizing改变盒模型的话，那么这个属性就决定了flex 元素的内容盒(content-box)的尺寸

flex-grow:[number]定义弹性盒子项（flex item）的拉伸因子
flex-shrink:[number]指定了 flex 元素的收缩规则

flex-flow: flex-direction 和 flex-wrap 的简写
flex-direction: row|row-reverse|column|column-reverse
flex-wrap:nowrap|wrap

order:规定了弹性容器中的可伸缩项目在布局时的顺序

/*对齐属性*/
align-content: 	适用于multi-line flex containers。设置了浏览器如何沿着伸缩盒子容器(flexbox container)的纵轴和网格容器(Grid Container)的主轴在内容项之间和周围分配空间。

align-items: 定义了浏览器如何分配顺着父容器交叉轴的弹性元素之间及其周围的空间。

align-self：属性 align-self 会对齐当前 flex 行中的 flex 元素，并覆盖已有的align-items 的值. 如果任何 flex 元素的侧轴方向 margin 值设置为 auto，则会忽略 align-self。In Grid, it aligns the item inside the grid area. 在Flexbox中会按照cross axis (当前flex 元素排列方向的垂直方向) 进行排列.

justify-content：定义了浏览器如何分配顺着父容器主轴的弹性元素之间及其周围的空间。

place-content

row-gap：行间距

column-gap：列间距

gap：row-gap和column-gap的简写

WARNING

order 仅仅对元素的视觉顺序 (visual order) 产生作用，并不会影响元素的逻辑或 tab 顺序。 order 不可以用于非视觉媒体，例如 speech。
